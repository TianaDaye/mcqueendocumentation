Firebase Realtime Database

Firebase Realtime Database - Documentation on Firebase


A Firebase Realtime Database is a cloud hosted database that stores JSON data and synchronizes it in real time to all connected clients. Unlike an HTTP request, whenever data changes, any connected device will receive the update in real time.


The GyroGoGo React application is connected to our GyroGoGo Car Locations Firebase Database to track our Gyrocars in real time. Due to real time data synchronization, each Gyrocar will be connected to the Firebase, along with the React application. All devices will have up to date location information, without delay. Firebase synchronization mechanism minimizes latency, making it ideal for tracking moving objects. It also scales automatically, with the ability to track anywhere from a few devices, to thousands of devices.


Our database contains a list of all cars, with their associated carID and coordinate location. The carID is used to connect our Firebase location information with our more static car and reservation data from our AWS-hosted SQL database. For example, when a car is added through our GyroGoGo React App, that carID is added to both the Firebase and SQL Database.


This database is tracking the car’s real location, and the SQL database tracks the cars ‘expected’ location, based on reservation and rental status. The actual and expected locations can be compared to ensure the car is in the right place, at the right time.

Testing - iOS Tracking App and GPX Simulation

Since the Gyrocars do not actually exist, we needed to simulate a moving car in order to test our location tracking and updates. Instead of connecting a real car GPS system to the Firebase, we simulated a car using an iPhone.

Test Car App

Getting the current location of a device | Apple Developer Documentation

Current location can be easily accessed through iOS CoreLocation on XCode. We created an app that constantly display’s the phone’s current coordinates. These coordinates update in live time. Each time there is a location change, the iOS app sends the coordinates to Firebase, based on the specified CarID. For testing purposes, we connected the app to Car 3.

Spoofing a Route

Simulating location in tests | Apple Developer Documentation

To test a car moving along a reservation path, we utilized GPX files in our app.


GPX files can simulate location data in an app for development and testing purposes. GPX stands for GPS Exchange Format, and they store location data, including latitude, longitude, altitude, and timestamps. They can simulate user location moving between various waypoints, or coordinate points, in a designated amount of time.


In our test app, there is a GPX file moving from one GyroGoGo station to another station. This simulated route will continuously loop while simulating. When this simulation is started, the iPhone thinks it is moving along this route, and it continuously updates it’s location and sends coordinates to Firebase.

Displaying the Car Location

In the Fleet Management page, there is a table displaying all car information, including the coordinates. As the coordinates change for Car 3 during location simulation, the coordinates update in real time in this table.
