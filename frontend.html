Navigation and User Authentication

This logic is complex, and we wanted to fully document it out.

BrowserRouter, Routes, Outlet, and Layout:


Layout.js

  • Contains conditional rendering

  • Each layout contains its respective nav bar, and an Outlet tag. The component defined in the associated route is ‘plugged in’ to the outlet tag.

  • The Route components are nested within a Layout component that manages the overall layout, including the header and is conditioned on the user's authentication and role.


UseAuth Hook - User Context

  • The useAuth hook accesses the authentication context, which provides user and logout functions.

  • This hook refreshes and re authenticates the user when necessary


Navigation.js

  • Called by App.js, nested within BrowserRouter, using React Router

  • It is the central routing mechanism for the entire application.

  • It ensures that the application can dynamically render different components based on the user's status and interactions with the app.

  • It handles:

    • User authentication

    • Role-based access control

    • Session management

    • User inactivity

  • UseNavigate controls navigation, and useLocation controls the current URL path.

  • A state employeeRoleNum stores the employee's role from session storage.

  • It can set an employee's role number (setEmployeeRoleNum) which affects the accessibility of various routes within the app based on their role.

  • An InactivityDetector component is used to log out the user after a specified period of inactivity.

  • Uses useEffect to update and recall the last location visited that isn't related to a reservation process. This is used for redirecting the user back to their last non-reservation-related page..

    • This is to ensure the page location persists on an event such as a page refresh

  • A set of Route components defines the URL paths and their corresponding React components.

    • These include Public Routes: Home, Login, Email Verification, Registration, and informational pages (FAQ, Contact, About Us).

    • Protected Routes: These routes require authentication and are wrapped in a ProtectedRoute component. They include user profiles and reservation-related pages.

    • Employee Routes: Wrapped in an EmployeeRoute component, these routes are accessible only to users with specific employee roles. They include dashboards and management tools for different aspects of the business, such as fleet, employees, customer approvals, etc.

  • Based on employeeRoleNum, certain routes will render either the intended component or a fallback Unauthorized component if the user's role does not grant them access.

  • If users access a path they have previously visited, they are redirected back to that path, ensuring they return to the last relevant context.

  • Defines a state for if the user is logged in, and a function to change this state (isLoggedIn).

  • Default route calls the Layout component with the isLoggedIn function as a property

    • This function will be used to pass the isLoggedIn state from child component (Login) to the parent component (Navigation)
